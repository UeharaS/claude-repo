# Schedule Tool

GoogleカレンダーをMCPツールで操作するスキル。カレンダーの読み書きに専念し、タスクのドメインロジックは持たない。

運用ルール（カラー、命名）はNotionのPJページ「スケジュールアシスタント」に集約されている。

**ナレッジページID:** `3325e790da8281e5b8e4ce13e2dc7203`

---

## ツール呼び出しのデフォルト値

| 項目 | 値 |
|------|-----|
| calendarId | `primary` |
| timeZone | `Asia/Tokyo` |
| sendUpdates | `none` |

---

## ユースケース一覧

### 1. 予定の確認

gcal_list_events で期間を指定して取得し、ユーザーにわかりやすくまとめる。

### 2. 予定の作成

**Notion参照:** notion-fetch → ナレッジページの「カラールール」「予定の入れ方ルール」セクション

取得したルールに従って、colorId・summary・locationを決定し、gcal_create_event で作成。

### 3. 予定の編集

gcal_list_events で対象のeventIdを取得し、gcal_update_event で更新。

カラーや命名の変更が伴う場合は、Notionの「カラールール」「予定の入れ方ルール」を参照。

### 4. 予定の削除

gcal_list_events で対象のeventIdを取得し、gcal_delete_event で削除。

### 5. 空き時間の検索

gcal_find_my_free_time で期間を指定して取得。

## スキルの責任範囲

| 操作 | 担当 |
|------|------|
| 予定の作成・編集・削除・確認 | schedule-tool（このスキル） |
| 空き時間の検索 | schedule-tool（このスキル） |
| タスクの管理・スケジューリング判断 | task-tool |
| タスクの作成・完了（Notion操作） | notion-workspace-manager |
| リマインドの管理 | notion-remind-manager |

---

## 注意事項

- タイムゾーンは必ず `Asia/Tokyo` を明示する（カレンダーのデフォルトがUTCのため）
- ドメイン知識はすべてNotionのPJページが正。SKILL.mdにはドメインルールのコピーを持たない
- タスクのカレンダーブロッキングはtask-toolが判断し、このスキルには「この名前で○時〜○時で登録して」という形で依頼が来る
